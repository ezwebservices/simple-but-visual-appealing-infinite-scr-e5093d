# Iteration Whiteboard

**Change request:** The actual app needs to be updated to num pals its currently named Math Scroll and the characters dont match we should use NumPals also I need aws amplify auth setup for parents to create an account and stripe setup to do a 14.99 per year subscription the create account process flow should handle this

**Subtasks planned:** 3

1. **Architect**: Design the parent account creation and subscription flow. Specify: (1) How Amplify Auth should be configured for parent accounts (email/password, required attributes like name, email verification). (2) Where Stripe Checkout fits in the signup flow — should the user create an Amplify account first then be redirected to Stripe Checkout for the $14.99/year subscription, or should payment happen during signup? (3) How to store subscription status (Amplify Data model or Stripe webhook to flag active subscriptions). (4) The Stripe webhook endpoint strategy (Lambda function URL or API route). (5) How the app checks subscription status to gate access. Produce a clear technical spec the Engineer can follow. Also audit all references to 'Math Scroll' or mismatched character names across src/ and config files and list what needs renaming to NumPals.
2. **Engineer**: Implement all three changes based on the Architect's spec: (1) REBRAND: Update all app references from Math Scroll to NumPals — check App.tsx, index.html, package.json, theme files, any hardcoded strings. Ensure character names in src/components/characters/ (BennyBear, LuluLadybug, OllieOwl, FifiFrog, ZiggyZebra, RexRobot) match the NumPals brand identity. (2) AMPLIFY AUTH: Expand amplify/auth/resource.ts to support parent account creation with email/password, add required user attributes (name, email). Build a signup/login UI flow in React (new AuthFlow component) that handles: create account → verify email → redirect to payment. Wire Amplify Auth into App.tsx so unauthenticated users see the auth flow. (3) STRIPE: Install @stripe/stripe-js, create a Stripe Checkout session flow for $14.99/year subscription. Add a Stripe webhook handler (Amplify Function or Lambda) to update subscription status. Gate app access behind active subscription with a 14-day free trial or immediate payment. Update amplify/backend.ts with any new resources needed.
3. **QA**: Test the complete account creation and subscription flow end-to-end: (1) Verify all Math Scroll references are gone — search entire codebase for old branding, confirm NumPals appears consistently. (2) Test parent signup: create account with email/password, receive and complete email verification. (3) Test Stripe integration: verify redirect to Stripe Checkout after account creation, confirm $14.99/year price displays correctly, test with Stripe test cards (success and failure scenarios). (4) Test subscription gating: confirm unauthenticated users cannot access the app, confirm unsubscribed users are prompted to pay. (5) Test login/logout flow works correctly. (6) Verify the app still functions correctly after auth (game loads, characters display, profiles work).
